# Product Scoping Brief — The Walcott & Co. Press

## Context Note

This is a NoDrft Systems proprietary product, not an external client engagement. "Discovery" in this context means product scoping — converting the homepage blueprint into a technically bounded build brief with confirmed stack, delivery phases, and Founder-approved scope. ## Objective

Convert the detailed Walcott homepage blueprint into a technically bounded build brief with confirmed scope, stack, and delivery phases. The Founder holds build authorization. No external client budget or timeline confirmation is required — product scope and delivery decisions are made by the Founder.

## Verified Facts

- Product: The Walcott & Co. Press (NoDrft Systems proprietary)
- Source document: `walcott_homepage_layout_blueprint.md` (handoff version)
- Project type: Authority website with content architecture, multiple conversion paths, and catalog layer
- Core positioning: "Walcott builds the publishing infrastructure that allows books to enter the market with audience, visibility, and conversion systems already in place."
- In-house assets: 10–15 manuscripts, 4 house authors
- Primary conversion paths: Service inquiry (Book a Strategy Call, Request a Project Review) + Publication interest (Buy on Amazon, Join release list)
- Recommended public nav: Home, Publications, Authors, Studio, Insights, About, Contact, Book a Strategy Call
- Homepage architecture: 14 sections (Hero → Trust Strip → Problem Diagnosis → Core Offers → How It Works → Proof → Featured Publications → Forthcoming Titles → House Authors → Lead Magnet → Insights Preview → Segmented Paths → Final CTA → Footer)
- SEO requirements: Title tag, meta description, canonical, Open Graph, Twitter cards, favicon, Organization schema, Book schema
- Mobile requirements: Persistent mobile nav, accessible CTAs, usable forms
- Technical requirement: Base page must remain readable if JS fails
