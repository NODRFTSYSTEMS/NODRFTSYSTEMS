## Post-Sweep Correction — 2026-05-03

**Finding:** On 2026-05-03, a `.claude/agents/` file count audit found that three agents in the 64-agent registry had no corresponding `.claude/agents/` definition files. These agents were present in `03_agent-skills/department-skill-pack/` (with SKILL.md, openai.yaml, and role-charter.md all populated) but their Claude Code activation files were absent. This was not caught by E-004 because E-004 checked the consolidation of activation records, not the file-by-file deployment completeness of the agents.

**Agents affected:**

| Code | Caribbean Name | Canonical Name | Department | Files Created |
|------|----------------|----------------|------------|---------------|
| AAA | Rochelle | Accessibility Audit Agent | Delivery & Build | `.claude/agents/aaa_accessibility.md` |
| BPA | Maritza | Bilingual Parity Agent | Quality & Compliance | `.claude/agents/bpa_bilingual_parity.md` |
| PLA | Simone | Plain Language Agent | Quality & Compliance | `.claude/agents/pla_plain_language.md` |

**Fix applied 2026-05-03:** Three `.claude/agents/` definition files created from their existing SKILL.md content. No new agents added to the registry — these agents were already authorized in the 64-agent bench; only their Claude Code activation files were missing.

**Root cause:** The agents were registered and had department-skill-pack anatomy (SKILL.md + openai.yaml + role-charter.md), but the `.claude/agents/` files were never created during the original batch deployment. The three reviewer agents with overlapping functions (reviewer_accessibility, reviewer_localization, reviewer_plain_language) may have been inadvertently treated as substitutes, masking the gap.
